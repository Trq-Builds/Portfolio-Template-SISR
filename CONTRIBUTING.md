# 🤝 Guide de contribution

Merci de votre intérêt pour ce projet ! Ce document définit les règles à suivre
pour proposer une contribution (issue, pull request).

---

## 🎯 Règles de priorité (P0 / P1 / P2)

| Niveau | Définition | Exemples |
|--------|------------|----------|
| **P0** | Bloquant – correction obligatoire avant merge | Régression fonctionnelle, faille XSS, donnée personnelle exposée |
| **P1** | Important – à corriger dans la version suivante | Bug visuel, accessibilité dégradée, performance anormale |
| **P2** | Nice-to-have – suggestions d'amélioration | Refactoring cosmétique, propositions de nouvelle feature |
